CSV parse functions

NAME

Extends gawk with facilities for parsing comma-separated-values (CVS) data.

SYNOPSIS

USAGE

@load "csv"
...
result = csvconvert(csvrecord, option...)
n = csvsplit(csvrecord, afield, option...)

DESCRIPTION

The csv gawk extension adds functions for parsing CSV data in a simple way.

csvconvert(csvrecord [, fs [, comma [, quote]]])
Returns the CSV formatted string argument converted to a regular awk record with fixed field separators. Returns a null string if csvrecord is not a valid string. The arguments are as follows:
csvrecord
The CSV formatted input string
fs
The resulting field separator. If omitted a null char '\0' is used.
comma
The actual CSV field delimiter. Default comma ','.
quote
The actual CSV quoting character. Default double quote '"'.
csvsplit(csvrecord, afield [, comma [, quote]]])
Splits the CSV formatted string argument into an array of individual fields and returns the number of fields. Returns -1 if csvrecord is not a valid string. The arguments are as follows:
csvrecord
The CSV formatted input string
afield
The resulting array of fields.
comma
The actual CSV field delimiter. Default comma ','.
quote
The actual CSV quoting character. Default double quote '"'.

EXAMPLES

Process CSV input records as arrays of fields:

{
    csvsplit($0, fields)
    if (fields[2]=="some value") print
}

Process CSV input records as awk regular records:

BEGIN {FS = "\0"}
{
    CSVRECORD = $0
    $0 = csvconvert($0)
    if ($2=="some value") print CSVRECORD
}

NOTES

BUGS

LIMITATIONS

Null characters are not allowed in fields. A null character terminates the record processing.
